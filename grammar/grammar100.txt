"""
The bash command synopsis specified here are slight modification from those in
the man pages (https://linux.die.net/man/), including the following syntactic
sugars:

    1. A flag name followed by "::" is interpreted as a sequence of single-letter
       flags; flags without any suffix are interpreted as an atomic flag.

       For example, "-fhLV::" -> "-f", "-h", "-L", "-V".

    2. An argument name followed by "..." (no space in between) is interpreted 
       as an argument list (i.e. one or more arguments are accepted). When a 
       special list separator (in general a non-alphabetic character) is used, it 
       is placed right before the "...". When no list separator is specified, the 
       default is ' '.

       For example, "pos,..." -> pos list separated by ",".

 Â   3. If an argument uses a specific regex pattern, the regex pattern is
       placed inside "<>" after the argument name.

       For example, "source<[[user@]host1:]file1>" -> the source is specified
       using format "[user@host1:]file1", where the "[user@host1:]" proportion
       is optional.

    2 & 3 can be used in composition.

    For example, "source<[[user@]host1:]file1>..." -> a list of source files
    specified using the required format.

    4. An embedded bash command is denoted using the name "command". Embedded
       programs in other languages are denoted using other names such as "program",
       "prog", etc.

       For example, the arguments to "which", "sudo" and "find -exec" are called
       "command", while the argument to "scp -S" is called "program", since it is 
       a DSL recognized by "scp".
"""
